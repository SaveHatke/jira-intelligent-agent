Role Descriptions & Permissions
Scrum Master

Can manage sprint capacity, view sprint health dashboard, predict spillovers, and perform quality analysis. Has access to management features.

Product Owner

Can create tickets, validate against DoR/DoD, break down epics, and generate release notes. Focuses on product management features.

Business Analyst

Can create tickets, validate requirements, and break down epics. Specializes in requirements analysis and ticket creation.

Tech Manager

Has administrative privileges including user management. Can access all features including SLA monitoring and system management.

Stakeholder

Can validate tickets, monitor SLAs, and access reporting features. Provides oversight and validation capabilities.

Note: All authenticated users have access to basic features like time machine, natural language queries, story estimation, ticket summarization, test generation, dependency mapping, and bug triage.
